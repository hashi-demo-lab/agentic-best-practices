Maximizing Throughput

Best Practices and Anti-Patterns for Subagent Performance

Best Practices
Right-size the model
Haiku for fast read-only exploration, Sonnet for balanced analysis, Opus only for complex reasoning. Match cost to task complexity.
3x faster 10x cheaper
Minimize return payload
Instruct subagents to return summaries, not raw data. Large returns exhaust parent context and trigger compaction.
90% less tokens Longer sessions
Scope narrowly
Specify files, directories, or patterns. Unbounded exploration is the most common anti-pattern that wastes tokens and time.
Targeted results Fewer iterations
Preload skills for domain knowledge
Use the skills field to inject conventions without runtime discovery overhead. Agent starts with the right context immediately.
Zero warmup Consistent output
Resume, don't restart
Subagents can be resumed by ID -- preserve context from previous work instead of starting fresh and re-discovering everything.
Context preserved 50% faster
Anti-Patterns
Using Opus for everything
Opus is powerful but slow and expensive. Most exploration and analysis tasks work fine with Haiku or Sonnet at a fraction of the cost.
10x cost overhead Slower response
Returning raw output to parent
Dumping full test logs, plan output, or file contents into parent context. This accelerates context exhaustion and compaction.
Context overflow Forced compaction
Unbounded exploration
Telling an agent to "investigate the codebase" with no scope. Agent wanders, burns tokens, and returns unfocused results.
Wasted tokens Unfocused output
Runtime convention discovery
Agent spends tokens reading files to learn patterns that could have been preloaded as skills. Wastes time on every invocation.
Repeated waste Slow start
Restarting from scratch
Creating a new subagent when you could resume an existing one. All previous context and progress is lost unnecessarily.
Context lost Duplicate work
