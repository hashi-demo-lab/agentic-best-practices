PLATFORM GUARDRAILS

Deny Permissions

Agents cannot modify their own verification tooling or guardrail configurations
Immutable guardrails — agents are denied write access to files that define their own verification and quality gates
Self-modification blocked — an agent cannot weaken linting, disable pre-commit hooks, or alter MCP server config
Scoped allow-list — explicit allow permissions grant only the commands agents need: terraform, git, pre-commit, MCP tools
Dangerous ops blocked — workspace variable creation and run execution require human approval via MCP deny rules
