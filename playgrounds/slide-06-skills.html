Skills: Domain Knowledge for Agents

SKILL.MD files package reusable expertise that the agent selects by matching name and description
Name and description — each SKILL.md declares a name and description in its frontmatter; the agent reads the description to decide when to use the skill
Domain injection — encode security baselines, coding patterns, workflow steps, and instructions as markdown files
Version-controlled — stored in .agent/skills/, shared via git, reusable across agents and projects
User-invocable option — skills can declare user-invocable: true so users launch them directly as slash commands
