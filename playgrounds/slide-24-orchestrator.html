The Orchestrator Pattern

Central coordinator dispatches agents, routes context via $ARGUMENTS
Never reads agent output -- orchestrator verifies files with Glob, never calls TaskOutput
Agents read from disk -- each agent loads its own inputs via the FEATURE path
All agents write to disk -- research, design, tests, and code are written as files; orchestrator never reads output directly
Glob verification after each agent -- confirm expected artifacts exist before next dispatch
$ARGUMENTS routing (FEATURE path + scope)
Disk-based artifact flow (design.md, code, tests)
Glob verification (artifact exists?)
