PLATFORM GUARDRAILS

Agent Identity & Role-Based Access Controls

Branch protection + project-scoped RBAC map every agent to a sandbox account boundary
How It Works
Dedicated team + token per agent — each agent identity is isolated with its own HCP Terraform team and API token
Agent team scoped to sandbox project — the team grants access only to the Agent Sandbox Project, never Prod/UAT/Dev
Branch protection enforces PR review — agent pushes to feature branches only, rulesets require human approval before merge
Dynamic credentials per project — project-level dynamic credentials map to an isolated cloud sub-account via OIDC
Why It Matters
No static secrets — credentials are short-lived and never exposed to the agent
Blast radius contained — scoped to sandbox project + sandbox cloud account only
Goal
Enable more agent autonomy by leveraging enterprise controls — workflow shifts to developer plan review, agent executes, developer code review
