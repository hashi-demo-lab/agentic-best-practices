PLATFORM GUARDRAILS

Agent Identity & Workload Federation

Zero long-lived credentials -- HCP Terraform manages the entire identity chain
Agent never sees cloud credentials -- the agent triggers runs and manages workspaces in HCP Terraform, nothing more
OIDC between workspace and provider -- HCP Terraform workspaces use workload identity federation to authenticate to AWS, Azure, GCP, or Vault
Dynamic credentials per run -- each Terraform run receives short-lived credentials via workspace trust relationships, eliminating secret sprawl
Blast radius contained -- project-scoped workspaces limit each agent to a single cloud account boundary
Key insight: The agent operates at the HCP Terraform API layer -- it never touches cloud provider credentials. The platform owns the identity chain end to end.
