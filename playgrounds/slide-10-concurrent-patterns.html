Concurrent Execution Patterns

Parallel Agent Strategies for IaC workflows

🔬
Fan-Out Research
Spawn multiple subagents to investigate independent areas simultaneously, then synthesize findings
Dispatch → Parallel Explore → Collect → Synthesize
Each agent explores a different area of the codebase or documentation. Parent collects summaries and synthesizes a unified view. Ideal for initial investigation of unfamiliar repos.
IaC Investigate modules, providers, and policies in parallel
🔍
Writer / Reviewer
Two parallel sessions with distinct roles — fresh context in reviewer prevents bias toward own code
Write ⇄ Review → Revise → Approve
Writer generates code, reviewer evaluates with fresh eyes. Separate contexts ensure the reviewer doesn't inherit the writer's assumptions. Catches issues a self-review would miss.
IaC One agent writes module, another reviews for compliance
⛓️
Subagent Chaining
Sequential delegation where each subagent output feeds the next — parent orchestrates context passing
Spec → Generate → Test → Document
Pipeline pattern where each stage transforms input for the next. Parent passes relevant output from one agent to the next. Each agent has clean context focused on its stage.
IaC Spec → Generate → Test → Document pipeline
📦
Isolate High-Volume Ops
Delegate test suites, log analysis, doc fetching — verbose output stays in subagent context
Delegate → Process (verbose) → Filter → Summary
Protect the parent context from token-heavy operations. Subagent processes verbose test output, log files, or API responses and returns only a concise summary.
IaC Run terraform plan in subagent, return only changes summary
🤖
Headless Mode
True process-level parallelism for batch processing, CI/CD pipelines, and file migrations
Trigger → Parallel Exec → Independent → Complete
Run multiple Claude instances as independent processes. No shared context — each operates on its own task. Best for batch operations that don't need coordination.
IaC Migrate multiple modules in parallel, each in its own process
