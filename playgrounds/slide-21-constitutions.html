SPEC-DRIVEN DEVELOPMENT

Constitutions: Non-Negotiable Rules

Machine-readable guardrails that govern every line of agent-generated code
Per-workflow guardrails -- each workflow has its own constitution with rules specific to its scope and risk profile
MUST/SHOULD/MAY severity -- structured RFC 2119 levels so agents know which rules are non-negotiable vs advisory
Deviation protocol -- weakening a security default requires a [SECURITY OVERRIDE] comment with documented justification
Version-controlled -- stored in .foundations/memory/, amendments require a pull request with human review
Module
module-constitution.md
MUST
encryption_enabled = true, public_access = false as defaults
.tftest.hcl written before module code (TDD)
No provider {} blocks -- modules inherit from consumers
Tags via merge() -- consumer tags take precedence
No static credentials (AWS_ACCESS_KEY_ID prohibited)
SHOULD
Mock-based tests for fast iteration
Cost-effective defaults -- consumers override for prod
Consumer
consumer-constitution.md
MUST
Private registry only -- raw resource blocks PROHIBITED
Version pin with ~> X.Y -- no latest or unconstrained
MUST NOT override module encryption/access defaults
HCP Terraform dynamic credentials -- no static keys
default_tags: ManagedBy, Environment, Project, Owner
SHOULD
Review CHANGELOG before version bumps
Avoid depends_on -- prefer explicit data flow
Provider
provider-constitution.md
MUST
Plugin Framework -- not SDKv2
types.String, types.Int64 -- never raw Go types
Test stubs compile before CRUD implementation
Delete silently succeeds on NotFound
Error messages never include credentials or tokens
SHOULD
stringvalidator.OneOf() for enum attributes
RequiresReplace() for ForceNew attributes
