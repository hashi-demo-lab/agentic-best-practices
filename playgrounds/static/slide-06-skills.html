Extending Claude with Domain Knowledge and Workflows

What are Skills?

Directories containing SKILL.md files with instructions, scripts, and resources that extend Claude's capabilities with domain-specific knowledge and workflows.
Definition
A directory with a SKILL.md file that defines instructions, scripts, and resources
Purpose
Extend Claude's capabilities with domain-specific knowledge and workflows
Key Feature
On-demand loading means skills don't consume context until needed
Analogy
Like plugins for an IDE -- installed but only activated when relevant
⚡

Progressive Disclosure

Claude loads skill content on-demand -- only metadata in context until triggered, enabling unbounded context bundling.
How It Works
Only metadata (name, description, triggers) loaded at startup
Trigger
Matching pattern detected (keyword, file type, user command)
Benefit
Dozens of skills registered without exhausting context window
Trade-off
Slight latency on first trigger, but dramatically better context efficiency
🎯

Skill Types

Three categories of skills, each serving a different purpose.
Reference Task Hybrid
Reference Conventions, coding patterns, style guides -- passive knowledge Claude consults when relevant
Task Step-by-step workflows with specific actions -- like a runbook Claude follows to complete a task
Hybrid Combine reference knowledge with task workflows -- e.g., Terraform module skill with conventions AND creation workflow
📦

Storage Locations

Skills stored at project, personal, or enterprise level with priority ordering.
Enterprise Organization-wide policies, security rules, compliance Highest priority
Personal ~/.claude/skills/ Individual preferences and workflows Mid priority
Project .claude/skills/ Shared via git, team-wide conventions Base priority
Progressive Disclosure Flow
Metadata Loaded Name, description, triggers registered
→

→
Full Content Injected SKILL.md loaded into context
→
Skill Executes Instructions, scripts, resources active
→
Context Preserved Unbounded bundling, no waste

Key Insight
On-demand loading means skills don't consume context until needed -- allowing unlimited skill bundling without exhausting the context window.

Best Fit for IaC
Hybrid skills excel for Terraform -- they encode naming conventions, compliance rules, AND module generation steps in a single skill.
🔍
Priority Order
Enterprise > Personal > Project -- higher priority skills override lower when they conflict, ensuring compliance at scale.
