Define project principles and rules that AI must follow. The foundational guardrail -- like a constitution for your codebase that every AI action must respect.
Generate user stories and acceptance criteria. Define what needs to be built and why. This becomes the source of truth guiding all subsequent AI work.
AI generates questions to eliminate ambiguity. Interactive back-and-forth ensures the spec is complete and unambiguous before planning begins.
Define tech stack, architecture, data models, and API contracts. Translates "what" into "how" with concrete technical decisions.
Systematic validation that the plan is complete. Checks UX, security, performance, accessibility, and quality dimensions before code is written.
Break the plan into atomic, testable work items with explicit dependencies. Each task is small enough for an agent to complete and verify independently.
Analyze output of completed tasks. Verify against specifications, run tests, measure quality, and feed learnings back into the process.

Summary SpecKit is GitHub's open-source toolkit for spec-driven development. Specs guide AI agents to generate, test, and validate code through 7 structured phases -- from project constitution to task execution. Each phase produces artifacts that feed into the next, creating a reliable chain from requirements to verified implementation.
