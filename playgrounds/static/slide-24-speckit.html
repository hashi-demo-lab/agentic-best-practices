PHASE 01
Constitution
"Project Principles"
Define project principles and rules that AI must follow. The foundational document that gates all subsequent work.
Artifacts
Coding standards Security requirements Naming conventions Forbidden patterns
PHASE 02
Specify
"What and Why"
Generate user stories and acceptance criteria. Define what needs to be built and the success metrics.
Artifacts
User stories Acceptance criteria Feature requirements Success metrics
PHASE 03
Clarify
"Eliminate Ambiguity"
AI generates questions to eliminate ambiguity. Resolve edge cases and conflicts before designing.
Artifacts
Edge cases Boundary conditions Integration points Error scenarios
PHASE 04
Plan
"Architecture & Design"
Define tech stack, architecture, data contracts and API contracts. Translate requirements into decisions.
Artifacts
Tech stack Architecture diagrams API contracts Data models
PHASE 05
Checklist
"Validate Completeness"
Systematically validate that the plan is complete -- CI, security, accessibility review.
Artifacts
CI review Security checklist Accessibility audit Completeness score
PHASE 06
Tasks
"Atomic Work Items"
Break the plan into atomic items with explicit dependencies. Each item small enough to verify independently.
Artifacts
Task dependencies Size criteria Acceptance tests Implementation order
PHASE 07
Analyze
"Verify & Learn"
Analyze completed task output, verify against specs, run tests, and feed learnings back into the process.
Artifacts
Test results Quality metrics Spec compliance Improvement areas
Summary SpecKit is the open-source model for spec-driven development. Specs guide AI agents to generate, test, and validate code through 7 structured phases — from project constitution to task execution. Each phase produces artifacts that feed into the next, creating a reliable chain from requirements to verified implementation.
