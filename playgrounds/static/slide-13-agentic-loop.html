1
🔎
Gather Context
Read the specification or task description to understand what needs to be done
Load relevant files and understand existing patterns before making changes
For IaC: read existing module structure, provider docs, policy constraints
2
✍️
Take Action
Generate Terraform resources, variables, outputs following conventions
Write test files alongside implementation (TDD for IaC)
Keep changes atomic and focused -- one logical change per iteration
3
✅
Verify Work
Rules-Based Feedback: Define clear output rules, then explain which rules failed and why
Terraform Validate: Syntax and configuration correctness
Terraform Plan: Preview infrastructure changes
TFLint: Best practice enforcement
Policy Checks: Security and compliance validation
↻
🔄
Self-Correct & Repeat
Agent reads error messages and rule violations to understand what went wrong
Makes targeted fixes based on specific feedback -- not random changes
Re-runs validation to confirm fix. Loop continues until all checks pass.
