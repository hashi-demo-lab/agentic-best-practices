Context Engineering

Best practices for structuring agentic IaC workflows
1
🎯 2-Tier Agent Architecture
Top-level orchestrator + specialist sub-agents
⚙
Pattern Top-level agent handles task tracking, orchestration initialization, and progress tracking
⚙
Pattern Specialist sub-agents handle execution and concurrency — isolated context, limited tools (least privilege and token efficient)
2
🔧 Initialization Scripts
Repeatable pre-requisites and context population
✓
Do Use initialization scripts for repeatable pre-requisites and context population
✓
Do Run validation checks — verify mandatory environment variables are set before starting
3
🛠️ Minimal Tool Enablement
Only enable required MCP tools
✗
Anti-pattern Enabling all tools "just in case" — unused tools waste context and risk instruction drift during compaction
✓
Do Scope each sub-agent to only the tools it actually needs — least privilege principle applies to agent tooling
4
💾 Task-Level Commits
Checkpoint progress with atomic commits per task
✓
Do Commit after each completed task — creates recovery points and clear audit trail
★
Benefit If a later task fails, the agent can revert to the last good state without losing all progress
5
📊 Context Window Management
Prevent compaction and context overflow
✓
Do Delegate verbose operations (test suites, log analysis) to sub-agents — output stays in their context
✓
Do Instruct sub-agents to return summaries, not raw data — large returns exhaust parent context
✗
Anti-pattern Unbounded exploration — always specify files, directories, or patterns to scope searches
