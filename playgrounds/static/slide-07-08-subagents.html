Subagent Architecture

14 Custom Agents Organized by Workflow

Orchestrator Dispatch Pattern

Module Agents

4 agents
tf-module-research opus
Answers ONE research question per instance (AWS docs, provider docs, registry)
tf-module-design opus
Produces specs/{FEATURE}/design.md with 7 mandatory sections
tf-module-test-writer opus
Converts design section 5 into .tftest.hcl files (TDD -- tests before code)
tf-module-developer opus
Implements ONE checklist item at a time, runs tests after each

Consumer Agents

3 agents
tf-consumer-research opus
Researches private registry modules, wiring patterns
tf-consumer-design opus
Produces consumer-design.md with 6 sections (module-first, NO raw resources)
tf-consumer-developer opus
Implements checklist items sequentially (module calls + wiring)

Provider Agents

5 agents
tf-provider-research opus
Researches API/SDK docs, Plugin Framework patterns
tf-provider-design opus
Produces provider-design-{resource}.md with 7 sections
tf-provider-test-writer opus
Creates Go test stubs before CRUD implementation
tf-provider-developer opus
Implements ONE checklist item in Go
tf-provider-validator opus
Runs fmt, validate, test, lint, security scan, generates report
Architecture Rules
Key Constraints
1
Isolated context windows -- each agent gets its own context, does NOT see parent conversation
2
No nesting -- subagents cannot spawn sub-subagents
3
Parent orchestrates all sequencing -- orchestrator skills control dispatch order
4
Custom system prompt per agent -- defined in .claude/agents/*.md
5
Receives CLAUDE.md files -- inherits project context automatically
6
Returns single message -- agent returns summary to parent, not raw output
Orchestration Rules
NEVER call TaskOutput for subagent results (except research agents)
Verify file existence with Glob after each agent -- DO NOT read contents into orchestrator
Downstream agents read their own inputs from disk
Research agents: parallel foreground calls, collect findings to pass to design
Minimal $ARGUMENTS -- only FEATURE path + scope
Agent Definition Format
--- name: tf-module-developer model: opus allowed_tools: [Read, Write, Edit, Bash, Glob, Grep, mcp__terraform__*, mcp__aws__*] --- (Markdown system prompt with role, skills, workflow steps, rules)
