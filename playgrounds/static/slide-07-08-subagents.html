Configuration

YAML Frontmatter

Define name, description, tools, model, permissions, skills, and hooks
--- name: terraform-module-gen description: Generate Terraform modules tools: [Read, Write, Bash, Glob, Grep] model: sonnet permissionMode: plan skills: [terraform-conventions] ---

System Prompt (Markdown Body)

Markdown content below frontmatter becomes the agent's system prompt
# Terraform Module Generator You are a specialist agent that generates compliant Terraform modules. ## Rules - Follow naming conventions - Include tests & documentation

Storage Locations

Project (.claude/agents/), User (~/.claude/agents/), CLI (--agents JSON), or Plugin (agents/ dir)

Management

Use /agents command for interactive management -- view, create, edit, configure tool access
Key Architecture Constraints
Isolated context windows
No nesting (can't spawn sub-subagents)
Parent orchestrates all sequencing
Custom system prompt per agent
Independent permissions
Receives CLAUDE.md files
Context Isolation Model
Context Each subagent runs in its own context window with a custom system prompt and specific tool access
No Nesting Subagents cannot spawn other subagents -- the parent must orchestrate all sequential or parallel calls
Receives System prompt + CLAUDE.md files + task description from parent. Does NOT see parent conversation history.
Returns A single message back to the parent with results. Verbose output stays contained in the subagent context.
Why Subagents
Speed -- Use faster models (Haiku) for read-only exploration tasks
Cost Control -- Scoped tools prevent expensive operations by lower-tier agents
Security -- Independent permissions and isolated context per agent
Modularity -- Reusable specialist agents across projects via shared configs
Orchestration -- Parent synthesizes results from multiple specialized agents
